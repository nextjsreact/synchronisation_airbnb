Introduction

Ce système extrait automatiquement toutes vos réservations Airbnb et les synchronise vers votre base de données Supabase, accessible depuis votre application Next.js déployée sur Vercel.

🕵️

CloakBrowser Stealth

Anti-detection navigateur (fingerprint, canvas, WebGL). Ne résout pas les CAPTCHAs — la session authentifiée les évite.

⚡

Sync < 10 minutes

Une nouvelle réservation Airbnb apparaît dans votre app en moins de 10 minutes.

☁️

Supabase Realtime

Votre dashboard Next.js se met à jour instantanément via websocket sans rechargement.

🐳

Docker complet

3 services Docker orchestrés : watcher iCal, scraper ciblé, scrape complet nocturne.

Architecture

Le système fonctionne en 3 couches indépendantes qui travaillent ensemble :

Couche 1 — toutes les 5 minutes
iCal Watcher ical_watcher.py
Vérifie si le calendrier d'une annonce a changé en téléchargeant son fichier iCal (simple requête HTTP, aucun navigateur). Si le hash change → ajoute dans la queue Supabase.
Couche 2 — déclenché par la queue (~2 min)
Targeted Scraper targeted_scraper.py
Lance CloakBrowser uniquement pour l'annonce modifiée. Récupère les détails financiers que l'iCal ne contient pas. Pousse immédiatement dans Supabase.
Couche 3 — toutes les nuits à 2h
Full Scraper airbnb_scraper.py
Scrape l'intégralité des 85 annonces. Sert de filet de sécurité pour rattraper tout ce qui aurait pu être manqué par les couches 1 et 2.
En continu — instantané
Supabase Realtime → Next.js
Dès qu'une donnée est insérée dans Supabase, votre application Next.js est notifiée via websocket et met à jour l'affichage sans rechargement.

Installation

Installer Docker Desktop

Téléchargez et installez Docker Desktop depuis docker.com.

Vérifiez l'installation :

docker --version
# Doit afficher : Docker version 24.x.x ou supérieur

Créer les tables Supabase

Dans votre dashboard Supabase → SQL Editor :

  1. Ouvrez le fichier sql/schema.sql
  2. Copiez tout le contenu
  3. Collez dans l'éditeur SQL Supabase
  4. Cliquez Run
✅
5 tables créées : reservations, listings, ical_hashes, sync_queue, sync_logs

Copier les fichiers sur votre serveur

# Sur votre VPS / serveur
mkdir ~/airbnb-scraper-v2
cd ~/airbnb-scraper-v2

# Copiez tous les fichiers du projet ici

Créer le fichier .env

cp .env.example .env
nano .env   # ou notepad .env sur Windows

Remplissez vos identifiants (voir section Configuration).

Construire les images Docker

docker compose build
# Durée : 5-10 min (télécharge CloakBrowser ~200MB)

Configuration

Toute la configuration se fait dans le fichier .env :

⚠️
Où trouver SUPABASE_SERVICE_KEY ?
Dashboard Supabase → Settings → API → section "Project API keys" → copier la clé service_role (pas la clé anon).
ℹ️
Comment obtenir TOTP_SECRET (Google Authenticator) ?
Airbnb → Compte → Connexion et sécurité → Authentification à 2 facteurs → Configurer → Scanner le QR code → Afficher le code secret → le copier.

Démarrage

Premier scrape complet

À faire une seule fois pour peupler toutes les tables :

docker compose run --rm full-scraper

Ce run effectue :

  • Connexion à Airbnb avec CloakBrowser
  • Scrape de toutes vos réservations
  • Collecte des URLs iCal de chaque annonce
  • Push dans Supabase (reservations + listings)
  • Export CSV + JSON local

Démarrer la surveillance temps réel

# Démarrer les 2 services en arrière-plan
docker compose up -d ical-watcher targeted-scraper

# Vérifier qu'ils tournent
docker compose ps
✅
À partir de maintenant, toute nouvelle réservation sera détectée et synchronisée en moins de 10 minutes.

Configurer le cron nocturne

crontab -e

# Ajouter cette ligne (adaptez le chemin) :
0 2 * * * cd ~/airbnb-scraper-v2 && docker compose run --rm full-scraper >> logs/full.log 2>&1

Intégrer dans Next.js

// lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)

// Écouter les nouvelles réservations en temps réel
supabase
  .channel('reservations')
  .on('postgres_changes', {
    event: '*',
    schema: 'public',
    table: 'reservations'
  }, (payload) => {
    // Rafraîchir votre state React ici
    console.log('Nouvelle réservation :', payload)
  })
  .subscribe()

Suivi dans Supabase

Vous pouvez suivre l'activité depuis votre dashboard Supabase :

-- Dernières synchronisations
SELECT type, status, reservations_count, duration_seconds, created_at
FROM sync_logs
ORDER BY created_at DESC
LIMIT 20;

-- Items en attente dans la queue
SELECT * FROM sync_queue WHERE status = 'pending';

-- Annonces sans URL iCal
SELECT listing_id, nom FROM listings WHERE ical_url = '' OR ical_url IS NULL;

# Scrape complet immédiat
docker compose run --rm full-scraper

Dépannage

SymptômeCause probableSolution
Aucune annonce dans le watcher Table listings vide Lancer d'abord docker compose run --rm full-scraper
iCal URL manquante pour une annonce Non trouvée lors du scrape Ajouter manuellement via SQL dans Supabase
2FA bloque en Docker TOTP_SECRET non configuré Ajouter TOTP_SECRET dans .env
Supabase : erreur 401 Mauvaise clé API Utiliser la clé service_role, pas anon
Container plante au démarrage Variable .env manquante Vérifier avec docker compose logs full-scraper
Next.js ne reçoit pas les updates Realtime non activé dans Supabase Dashboard Supabase → Database → Replication → activer reservations
